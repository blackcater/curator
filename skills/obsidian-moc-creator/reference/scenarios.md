## Learning

**When to use**: User wants to systematically learn a new technology or concept.

**Goal**: Create a knowledge map that guides learning from basics to proficiency.

**Structure**:
```
主题名称
├── 入门导览
│   ├── 什么是 [主题]
│   ├── 为什么需要 [主题]
│   └── 学习路线概览
├── 核心概念
│   ├── 概念 A
│   ├── 概念 B
│   └── 概念 C
├── 进阶主题
│   ├── 高级概念 D
│   └── 底层原理
├── 实践项目
│   ├── 入门项目
│   └── 进阶项目
├── 常见问题
│   ├── FAQ 1
│   └── FAQ 2
├── 学习资源
│   ├── 官方文档
│   ├── 优质文章
│   └── 开源项目
└── 关联主题
    ├── 相关主题 A
    └── 相关主题 B
```

**Trigger keywords**: "学习", "入门", "掌握", "教程", "系统学习"

## Interview

**When to use**: User is preparing for technical interviews.

**Goal**: Create a comprehensive interview preparation map.

**Structure**:
```
[技术栈] 面试准备
├── 基础知识
│   ├── 核心概念
│   ├── 基础原理
│   └── 常见问题
├── 进阶知识
│   ├── 高级特性
│   ├── 性能优化
│   └── 源码原理
├── 项目实战
│   ├── 项目经验总结
│   ├── 场景题解答
│   └── 系统设计
├── 编码练习
│   ├── 经典题目
│   └── 变形题

```

**Trigger keywords**: "面试", "准备", "刷题", "面经", "P6", "P7", "高级"

## Project

**When to use**: User needs to document or reference a project implementation.

**Goal**: Create a project knowledge map for future reference and team sharing.

**Structure**:
```
[项目名称] 技术文档
├── 项目概述
│   ├── 背景与目标
│   ├── 技术栈概览
│   └── 核心功能
├── 架构设计
│   ├── 系统架构
│   ├── 模块划分
│   └── 数据流设计
├── 技术选型
│   ├── 为什么选择 A
│   ├── 为什么选择 B
│   └── 替代方案对比
├── 最佳实践
│   ├── 代码规范
│   ├── 设计模式
│   └── 性能优化
├── 踩坑记录
│   ├── 问题描述
│   ├── 解决方案
│   └── 经验总结
├── 运维相关
│   ├── 部署方案
│   ├── 监控告警
│   └── 故障排查
└── 扩展阅读
    ├── 相关文档
    └── 参考资料
```

**Trigger keywords**: "项目", "技术方案", "架构", "文档", "踩坑"

## Depth Levels

For each scenario, adjust detail level:

- **Beginner**: Focus on fundamentals, skip advanced topics
- **Intermediate**: Balance basics and advanced content
- **Advanced**: Comprehensive, includes expert-level topics
